## O que é um ADR?

Um ADR registra uma decisão arquitetural relevante: **contexto**, **decisão tomada**, **alternativas consideradas** e **consequências**. Serve como memória técnica do projeto.

## Quando criar um ADR?

- Adoção ou mudança de tecnologia/framework
- Escolha de padrão de comunicação entre serviços
- Decisão de modelagem de dados com impacto cross-domínio
- Qualquer decisão que seja difícil ou custosa de reverter

## Template

```markdown
# ADR-XXXX — Título da Decisão

**Data:** AAAA-MM-DD
**Status:** Proposto | Aceito | Depreciado | Substituído por ADR-XXXX
**Autor:** <agente ou humano>

## Contexto
Descrever o problema ou situação que motivou a decisão.

## Decisão
Descrever a decisão tomada de forma clara e objetiva.

## Alternativas Consideradas
- Alternativa A: ...
- Alternativa B: ...

## Consequências
- Positivas: ...
- Negativas / trade-offs: ...
- Débitos gerados (se houver): registrar em `contexts/quality/tech-debt.md`
```
